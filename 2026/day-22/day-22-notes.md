# Day 22 Notes

## Answers

1. `git add` stages changes; `git commit` records staged changes in history.
2. The staging area lets you choose exactly what goes into the next commit.
3. `git log` shows commit hashes, authors, dates and messages.
4. `.git/` stores repository metadata and history.
5. Working directory is files on disk, staging area is the next commit draft, repository is committed history.
