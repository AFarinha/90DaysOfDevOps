# Day 32 Volumes and Networking

## The Problem

Without a volume, data written inside a container disappears when the container is removed.

## Named Volumes

Use named volumes for Docker-managed persistent data, especially databases.

## Bind Mounts

Use bind mounts when a local host directory should be visible inside a container.

## Custom Networks

Custom bridge networks provide container-name DNS, so services can reach each other by name.
