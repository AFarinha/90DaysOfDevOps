# Tasks - Day 03

Use these commands as a practical checklist for the day. Replace placeholders such as `<...>` before running them.

## Commands

| Command | What it does |
|---|---|
| `cd 2026/day-03` | Changes into the target directory before running the exercise commands. |
| `touch linux-commands-cheatsheet.md` | Creates one or more empty files if they do not already exist. |
| `less linux-commands-cheatsheet.md` | Opens a file in a pager for reading. |
| `ps aux` | Lists running processes with the requested format. |
| `top` | Opens a live view of CPU, memory and process activity. |
| `pgrep ssh` | Finds process IDs matching the provided process name. |
| `ls -lah` | Lists files, directories or metadata such as permissions and ownership. |
| `find . -name "*.md"` | Runs this step as part of the day exercise; review the surrounding task before executing it. |
| `df -h` | Shows filesystem disk usage. |
| `du -sh .` | Shows disk usage for files or directories. |
| `ping -c 4 google.com` | Tests basic network reachability to the target host. |
| `ip addr show` | Shows network interfaces and assigned IP addresses. |
| `dig google.com` | Queries DNS records for the target domain. |
| `curl -I https://example.com` | Makes an HTTP request or checks HTTP headers for the target URL. |
| `journalctl -n 50` | Reads logs from journald, optionally filtered by service and line count. |
