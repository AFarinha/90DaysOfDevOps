# Day 23 Notes

## Branches

- A branch is a movable pointer to a commit.
- Branches keep feature work separate from `main`.
- `HEAD` points to the current branch or commit.
- Switching branches updates the working tree to match that branch.

## GitHub

- `origin` is the default remote name for your repository.
- `upstream` is commonly used for the original repository after forking.
- `git fetch` downloads remote changes without merging.
- `git pull` is fetch plus integration into the current branch.
- Clone copies a repository; fork creates a server-side copy under your GitHub account.
