# Tasks - Day 44

Use these commands as a practical checklist for the day. Replace placeholders such as `<branch-name>` before running them.

## Commands

| Command | What it does |
|---|---|
| `cd 2026/day-44/github-actions-practice` | Changes into the GitHub Actions practice repository. |
| `cat .github/workflows/secrets-artifacts-tests.yml` | Prints the workflow that demonstrates secrets, artifacts, caching and tests. |
| `cat scripts/math_tools.py` | Prints the Python module used by the CI tests. |
| `cat tests/test_math_tools.py` | Prints the pytest test file used by the workflow. |
| `python -m pip install -r requirements.txt` | Installs local Python test dependencies. |
| `python -m pytest -q` | Runs the tests locally before pushing to GitHub. |
| `git status` | Shows changed and untracked files before committing. |
| `git add .github/workflows/secrets-artifacts-tests.yml scripts/math_tools.py tests/test_math_tools.py requirements.txt` | Stages the Day 44 workflow, script, tests and dependencies. |
| `git commit -m "Add secrets artifacts and test workflow"` | Creates a commit with the Day 44 CI work. |
| `git push` | Pushes the workflow changes to GitHub so Actions can run them. |
| `gh run list` | Lists recent workflow runs to confirm the CI run started. |

## GitHub UI Setup

These steps are done in the GitHub web UI, not in the terminal.

| Step | What it does |
|---|---|
| `Settings -> Secrets and variables -> Actions -> New repository secret -> MY_SECRET_MESSAGE` | Creates the secret used by the workflow to check whether a secret exists. |
| `Settings -> Secrets and variables -> Actions -> New repository secret -> DOCKER_USERNAME` | Creates a Docker Hub username secret for a future Docker publishing workflow. |
| `Settings -> Secrets and variables -> Actions -> New repository secret -> DOCKER_TOKEN` | Creates a Docker Hub token secret for a future Docker publishing workflow. |
| `Actions -> workflow run -> Artifacts -> Download` | Downloads the uploaded artifact generated by the workflow. |
