# Notes - Day 38

Summary: Learn YAML basics before writing CI/CD pipeline files.

- YAML stores structured data using indentation, key-value pairs, lists and nested objects.
- YAML uses spaces, not tabs.
- Block strings with `|` preserve line breaks; folded strings with `>` combine lines into a paragraph.
- Booleans such as `true` and `false` are different from quoted strings such as `"true"`.
- Validation catches indentation and syntax errors before YAML is used by tools such as Docker Compose, Kubernetes or CI/CD systems.
