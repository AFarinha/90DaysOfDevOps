# Day 18 Scripting

## Scripts

- `functions.sh`: basic functions with arguments.
- `disk_check.sh`: disk and memory checks.
- `strict_demo.sh`: strict mode demonstration.
- `local_demo.sh`: local vs global variables.
- `system_info.sh`: system information reporter.

## `set -euo pipefail`

- `set -e`: exits on command failure.
- `set -u`: fails on unset variables.
- `set -o pipefail`: fails a pipeline if any command in it fails.

## What I Learned

- Functions make scripts easier to read.
- Strict mode is useful for operational scripts.
- Local variables avoid accidental state leaks.
