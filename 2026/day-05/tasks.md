Use these commands as a practical checklist for the day. Replace placeholders such as `<...>` before running them.

## Commands

| Command | What it does |
|---|---|
| `cd 2026/day-05` | Changes into the target directory before running the exercise commands. |
| `uname -a` | Prints kernel and system information. |
| `cat /etc/os-release` | Prints file contents to the terminal. |
| `mkdir -p /tmp/runbook-demo` | Creates the requested directory path. |
| `cp /etc/hosts /tmp/runbook-demo/hosts-copy` | Copies files or directories from source to destination. |
| `ls -l /tmp/runbook-demo` | Lists files, directories or metadata such as permissions and ownership. |
| `free -h` | Shows memory and swap usage. |
| `df -h` | Shows filesystem disk usage. |
| `du -sh /var/log` | Shows disk usage for files or directories. |
| `ss -tulpn` | Lists sockets and listening services. |
| `ping -c 4 google.com` | Tests basic network reachability to the target host. |
| `journalctl -u ssh -n 50` | Reads logs from journald, optionally filtered by service and line count. |
| `tail -n 50 /var/log/syslog` | Shows the last lines of a file, commonly used for logs. |
