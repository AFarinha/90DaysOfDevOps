# Notes - Day 39

Summary: Understand CI/CD concepts before writing pipeline files.

- CI/CD reduces the risk of manual integration and deployment work.
- Continuous Integration catches issues early by building and testing code frequently.
- Continuous Delivery keeps software ready to deploy, usually with a manual approval step.
- Continuous Deployment automatically deploys successful changes to production.
- A pipeline is made of triggers, stages, jobs, steps, runners and artifacts.
