# Notes - Day 42

Summary: Understand GitHub-hosted and self-hosted runners.

- A runner is the machine that executes a GitHub Actions job.
- GitHub-hosted runners are created, patched and destroyed by GitHub for each job.
- Self-hosted runners are machines that you own or manage and register with a repository, organization or enterprise.
- Pre-installed tools matter because they reduce setup time in jobs.
- Labels help target specific self-hosted runners when different machines have different operating systems, tools or permissions.
